# 技术局限

## AI 模型局限

### 大语言模型 (LLM)
- 可能产生"幻觉"——生成听起来合理但实际错误的内容
- 知识截止日期取决于所用模型版本
- 不保证医学回答的完全准确性

### 医学影像模型
- **脑肿瘤检测**：基于 U-Net 架构，训练数据集有限，可能对罕见肿瘤类型不敏感
- **胸部 X 光**：仅区分 COVID-19 与正常，不能检测其他肺部疾病（如肺结核、肺癌）
- **皮肤病变**：仅做分割，不能独立给出诊断结论
- 模型推理结果受图像质量影响很大

## 系统局限

### 上下文窗口
- LLM 有最大上下文长度限制，超长对话可能导致早期信息丢失
- 当前上下文管理仅按消息条数截断，无语义摘要策略

### 意图识别
- 当前使用关键词匹配，而非 LLM 语义理解
- 可能误判用户意图（如将闲聊识别为健康咨询）

### 工具调用
- 当前使用硬编码路由，Agent 不能自主决定调用哪些工具
- 不能组合使用多个工具（如先搜索再检索再分析）

### 个人健康记忆
- 健康记录中 3 年前的数据与最近数据权重相同
- 对话中提到的重要信息（如"我父亲有糖尿病"）不会被自动提取为长期记忆
- 模糊语义匹配（如"最近容易累"匹配"贫血史"）依赖 embedding 质量

## 部署局限

### 资源需求
- Qdrant 向量数据库需要额外存储空间
- 三个 CV 模型（.pth/.pt 文件）约需 500MB+ 磁盘空间
- LLM API 需要稳定的网络连接

### 平台支持
- 目前仅提供 Web 界面
- 无移动端 App
- 无离线模式

## 法规局限

- 本系统**不是医疗器械**，未获得 FDA/CE/NMPA 等任何监管机构批准
- AI 生成的医学内容不应被视为医疗建议
- 用户对使用本系统产生的任何后果自行承担责任
